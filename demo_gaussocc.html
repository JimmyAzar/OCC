Here I implement a classical normal-based one-class classifier. To test it out, I generate an artificial 2D dataset consisting for two Gaussian blobs at a distance from each other forming the target class. Since the classifier will try to cover the target class with a single Gaussian distribution, (outlier) objects falling in between the two blobs will be falsely classified as inliers. Thus, this dataset shows to some extent the limitation of the Gaussian one-class classifier.

Gaussian One-Class Classifier

The target class is assumed to follow a multivariate Gaussian distribution with probability density function:

\[f_X(x) = \dfrac{1}{(2 \pi)^{p/2} |\Sigma|^{1/2}} exp\left( \dfrac{-1}{2} (x-\mu)^T \Sigma^{-1} (x-\mu) \right)\]

The mean \(\mu\) and covariance matrix \(\Sigma\) are sample estimates as obtained from the training set.

A threshold \(\theta\) is defined to reject a certain fraction of the training set: fracrej \(\in [0,1]\). This is the only hyperparameter of the classifier, and it is optimized using an ROC curve.

In addition to returning hard labels based on whether the estimated output density is above the threshold (inlier) or below the threshold (outlier), I have modified the classifier to return a probability-like output as well by considering only the exponential part of the above density function, i.e.

\[p(x) = exp\left( \dfrac{-1}{2}(x-\mu)^T \Sigma^{-1} (x-\mu) \right)\] Having a probability like output in the range [0,1], allows for combining this classifier with other one-class classifiers with similar probability output measures.

To obtain hard labels, the classification rule becomes:

\[L(x) = \begin{cases} target, & \text{if } p(x) \geq \theta \\ outlier, & \text{if } p(x) < \theta \\ \end{cases} \]

Dataset Generation

As toy dataset, I generate two circularly symmetric Gaussian distributions in 2D with zero mean and with the identity matrix as covariance matrix. The training set consists only of inliers. The test set may contain inliers and outliers. I have drawn the outliers randomly from a uniform distribution. Inliers are assumed to have a ‘0’ label while outliers have a ‘1’ label.

gausscirc <- function(ntrn_inlier=100, ntst_inlier=20, ntst_outlier=20, show=TRUE, seed=1){
#Generates two circularly symmetric Gaussian distributions in 2D with zero mean and with the identity matrix as covariance matrix. 
#The training set consists only of inliers; their number is ntrn_inliers objects per distribution.
#The test set may consists of inliers (ntst_inlier objects per distribution) and should consist of outliers (ntst_outlier objects per distribution) drawn randomly from a uniform distribution.
#Inliers are assumed to have a '0' label while outliers have a '1' label. 
  
  if (!require("MASS")) install.packages("MASS") 
  library("MASS")

  #generate training inliers
  set.seed(seed)
  x <- 0.3*mvrnorm(ntrn_inlier, mu = c(0,0), Sigma = matrix(c(1,0,0,1),nrow = 2))
  xin <- rbind(x+2,x-2)
  xin <- cbind(xin,rep(0,ntrn_inlier)) #add '0' labels for inliers
  
  #generate test inliers (if any)
  if (ntst_inlier!=0){
    set.seed(seed+1)
    z <- 0.3*mvrnorm(ntst_inlier, mu = c(0,0), Sigma = matrix(c(1,0,0,1),nrow = 2))
    zin <- rbind(z+2,z-2)
    zin <- cbind(zin,rep(0,ntst_inlier)) #add '0' labels for inliers
  }
  else{zin <- numeric(0)}
  
  #generate test outliers from uniform distribution between [-5,5]
  set.seed(seed)
  f1 <- runif(ntst_outlier, min = -5, max = 5) #uniform distribution for feature 1
  set.seed(seed+1)
  f2 <- runif(ntst_outlier, min = -5, max = 5) #uniform distribution for feature 2
  zout <- cbind(f1,f2,rep(1,ntst_outlier)) 
  
  xtrn <- as.data.frame(xin)
  colnames(xtrn) <- c("f1","f2","label")

  xtst <- as.data.frame(rbind(zin,zout)) 
  colnames(xtst) <- c("f1","f2","label")

  if(show){
    plot(xin, col="blue", pch=20, xlab = "feature 1", ylab = "feature 2",
         xlim = c(-5.2,5.2), ylim = c(-5.2,5.2))
    points(zin,pch=20,col="violetred")
    points(zout,pch=17,col="violetred")
    legend("bottomright",legend = c("training inlier","test inlier","test outlier"),
           col = c("blue","violetred","violetred"), pch = c(20,20,17), cex=0.8)
  }
  
  l = list(xtrn,xtst) #create list of outputs
  return(l)
}

Classifier

The function gaussocc() implements the Gaussian one-class classifier and returns predicted hard and soft labels of the test set objects. As input, the training set contains only examples of inliers (target objects), while the test set may contain both inliers and outliers. The rejection fraction is by default set at 5%, which will correspond to a threshold \(\theta\) such that the training objects in the lowest 5% tail are cut off as outliers. This hyperparameter, fracrej can be optimized automatically using an ROC curve as demonstrated in a later section.

gaussocc <- function(xtrn, xtst, fracrej = 0.05){
  #Training set contains only inliers. 
  #Fracrej is the fraction of training objects to reject
  
  p <- ncol(xtrn)-1 #number of features
  n <- nrow(xtrn) #numer of training objects
  mu <- apply(xtrn[,-ncol(xtrn)],2,mean) #overall mean
  covar <- cov(xtrn[,-ncol(xtrn)]) #covariance matrix  
  X <- xtrn[,-ncol(xtrn)] - matrix(rep(mu,n),nrow=n,byrow=TRUE)
  D <- (as.matrix(X) %*% solve(covar))*X
  d <- apply(D,1,sum)
  
  #Relative densities of training objects
  ftrn <- 1/sqrt((2*pi)^p * det(covar)) * exp(-0.5*d) #densities
  Ptrn <- exp(-0.5*d) #probability of being target
  
  #Relative densities of test objects
  n <- nrow(xtst) #number of test objects
  Z <- xtst[,-ncol(xtst)] - matrix(rep(mu,n),nrow=n,byrow=TRUE) #using mean of training set
  Dz <- (as.matrix(Z) %*% solve(covar))*Z #using covariance of training set
  dz <- apply(Dz,1,sum)
  #S <- cov(xtst[,-ncol(xtst)])
  ftst <- 1/sqrt((2*pi)^p * det(covar)) * exp(-0.5*dz) #densities
  Ptst <- as.numeric(exp(-0.5*dz)) #probability of being target 
  
  #Compute threshold equivalent to fracrej
  nr <- length(Ptrn) #number of target/training objects
  nobj_acc <- round((1-fracrej)*nr)
  po <- sort(Ptrn, decreasing = TRUE)
  if (nobj_acc==0) thr <- 1
  else if (nobj_acc==nr) thr<- 0
  else thr <- po[nobj_acc]
  
  #Label test set
  Ltst <- rep(0,nrow(xtst))
  Ltst[Ptst < thr] <- 1 #assign as outlier
  
  l <- list(Ltst,Ptst)
  return(l)
}

Visualization of decision boundary

For visualizing the decision boundary of the Gaussian OCC (and other classifiers) in 2D, I have written the below function viz() which plots the training (inlier) objects, then calls the Gaussian classifier which is trained on these objects in order to classify a mesh grid of points covering the range of the training set in both x- and y- directions. Each grid point is classified as either outlier or inlier and is plotted in different color for each case. In addition, the boundary that separates inliers from outliers is plotted as a contour line in black.

viz <- function(data, resolution = 100, classifier, ...){
  #visualize decision boundary
  #data is labeled 2D dataset with labels as 3rd (last) column
  #grid resolution is 100 points per axis by default 
  
  cl <- data[,3] #class labels; normally just training objects with label '0'
  kk <- length(unique(cl)) #number of classes
  color <- c("blue","violetred") 
  shape <- c(20,17) #dot, triangle
  
  #find x-, y- limits of data
  mn <- apply(data[,1:2],2,min)
  mx <- apply(data[,1:2],2,max)
  xmin <- mn[1]; ymin <- mn[2]
  xmax <- mx[1]; ymax <- mx[2]
  low <- min(xmin,ymin)-3 #add margin
  high <- max(xmax,ymax)+3 #add margin
  
  #plot data objects in blue
  plot(data[,1:2],col = color[1], pch = shape[as.integer(cl)+1L],
       xlim = c(low,high),ylim = c(low,high),xlab("Feature 1"),ylab("Feature 2"))
  
  #set-up grid
  xs <- seq(low, high, length.out = resolution)
  ys <- seq(low, high, length.out = resolution)
  g <- cbind(rep(xs, each=resolution), rep(ys, time = resolution))
  g <- cbind(g, rep(0,nrow(g))) #add dummy labels
  g <- as.data.frame(g)
  colnames(g) <- colnames(data)
  
  #call classifier to predict grid point labels
  if (classifier == "gauss")  out <- gaussocc(data,g,...)
  else if (classifier == "parzen") out <- parzenocc(data,g,...)
  else if (classifier == "knn") out <- knnocc(data,g,...)
  else if (classifier == "rknnd") out <- rknndocc(data,g,...)
  else if (classifier == "kmeans") out <- kmeansocc(data,g,...)
  else if (classifier == "som") out <- somocc(data,g,...)
  else if (classifier == "gmm") out <- gmmocc(data,g,...)
  else message("Classifier specified is unknown!")
  L <- out[[1]]

  #plot grid points in colors reflecting their predicted labels
  points(g, col = color[as.integer(L)+1L], pch = ".")
  
  #add decision boundary contour
  z <- matrix(L+1L, nrow = resolution, byrow = TRUE)
  contour(xs, ys, z, add = TRUE, drawlabels = FALSE,
          lwd = 2, levels = (1:(kk-1))+.5)
}

Demo

In this demo, I use all the above functions to show the resulting output for the artificial dataset consisting of two separate Gaussian blobs.

First, I call gausscirc() to generate a dataset.

l <- gausscirc(ntrn_inlier = 200, ntst_inlier =  100, ntst_outlier = 100, 
               show = TRUE, seed=1)

I split the original test set into a validation set (for tuning the hyperparameter fracrej) and a final test set for assessing the performance of the classifier. The split I have selected is a random 50%-50% stratified split.

xtrn <- l[[1]]
xtst <- l[[2]]
#Split test set into validation set and final test set.
inliers <- which(xtst[,ncol(xtst)]==0)
outliers <- which(xtst[,ncol(xtst)]==1)
ind_in <- sample(inliers,floor(0.5*length(inliers))) #50% stratified split
ind_out <- sample(outliers,floor(0.5*length(outliers))) 
xval <- xtst[c(ind_in,ind_out),]
xtst <- xtst[-c(ind_in,ind_out),]

For each possible value of fracrej in the range [0,1], train the classifier accordingly and compute the classification errors for inliers and outliers of the validation set.

#Optimize fracrej parameter using validation set
R <- seq(1,0,length.out = 500) #fracrej in [0,1]
error_inliers <- error_outliers <- numeric(0) 
for (i in seq_along(R)){
  l <- gaussocc(xtrn,xval,fracrej = R[i])
  L <- l[[1]]
  error_outliers[i] <- sum(L==0 & xval[,ncol(xval)]==1)
  error_inliers[i] <- sum(L==1 & xval[,ncol(xval)]==0)
}

Plot a receiver-operating characteristic (ROC) curve and select the optimal value for fracrej which corresponds to the point on the curve that is closest to the upper left corner of (0,1), i.e. perfect classification.

#Plot ROC curve
y <- 1 - (error_inliers / sum(xval[,ncol(xval)]==0)) #frac targets accepted
x <- (error_outliers / sum(xval[,ncol(xval)]==1)) #frac outliers accepted             
plot(x,y,type="l",lwd=2,xlab="fraction outliers accepted",ylab="fraction targets accepted") #ROC curve
    
#Find optimal point on ROC closest to upper left corner i.e. (0,1)
index <- which.min(x^2 + (y-1)^2)
points(x[index],y[index],pch=1,col="red",lwd=2)

fracrej_opt <- R[index] #optimal fracrej 
paste("Optimal fracrej =",fracrej_opt)
## [1] "Optimal fracrej = 0.11623246492986"

Use the optimal value of fracrej to call the classifier once more and evaluate its performance on the final test set using a confusion matrix and the measures derived from it such as sensitivity, specificity, etc.
